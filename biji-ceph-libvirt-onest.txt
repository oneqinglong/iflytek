【开发目标】

开发目标是简单定义为以下3项的: 
1.可以轻松扩展到数PB容量
2.支持多种工作负载的高性能(每s输入和输出操作IOPS和带宽)
3.高可靠性

但是这些目标质检会互相竞争的
例如说可扩展性会降低或者抑制性能或者影响可靠性的
ceph的设计还包括保护单点故障的容错能力
它假设大规模(PB级别存储)存储故障是常见现象不是例外情况
它的设计并没有假设某种特殊工作负载
但是包括了适应变化的工作负载
并提供了最佳性能的能力
它利用POSIX的兼容性完成所有这些任务
允许它对当前依赖POSIX语义的应用进行透明的部署。
通过以ceph为目标的改进。

【系统架构】

ceph生态系统架构可以划分为四个部分的。
1.clients-客户端(数据用户)
2.cmds-metadata server cluster
元数据服务器(缓存和同步分布式元数据)
3.cosd-object storage cluster
对象存储集群(将数据和元数据作为对象存储，执行其他关键职能)
4.cmon-clustor monitors
集群监视器(执行监视功能)

☆ libvirt

libvirt是一套免费开源的支持【linux下主流虚拟化工具】的C函数库
libvirt是C函数库的
libvirt是提供了一套方便可靠的编程接口的。
是支持C/C++/RUBY/PYTHON的。
libvirt是为【Xen,kvm】在内的各种虚拟化工具
提供一套方便可靠的编程接口
支持与C/C++/Ruby/python/java等多种主流开发语言的绑定
当前主流linux平台上默认的虚拟化管理工具
【virt-manager图形化】
【virt-install命令行模式】
都是基于libvirt开发而成

libvirt库是一种实现linux虚拟化功能的linux api
它支持各种【虚拟机监控程序】
包括【xen/kvm/qemu】
和用于其他操作系统的一些虚拟产品

libvirt提供一种虚拟机监控程序不可知的API
来安全管理运行于主机上的客户操作系统
libvirt本身不是一种工具
它是一种可以建立工具来管理客户操作系统的API
libvirt本身构建于一种抽象的概念之上
它为受支持的虚拟机监控程序实现的常用功能提供通用的API
libvirt起初是专门为Xen设计的一种管理API
后来被扩展为可支持多个虚拟机监控程序
